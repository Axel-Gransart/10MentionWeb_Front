Mon premier formulaire en HTML

Le formulaire sert à récupérer des données que l'utilisateur va entrer et les envoyer au serveur pour un traitement (en PHP par exemple): login, formulaire de contact et même le chat facebook.

Je déclare le formulaire avec la balise form: <form> est une balise essentielle, elle permet d'indiquer le début d'un formulaire, dont on va se servir énormément en developpement. Cette balise fonctionne avec 2 attributs essentiels:

  • - action: permet d'indiquer où on récupère les données envoyées par l'utilisateur, l'adresse de la page ou du fichier qui va traiter les informations envoyées par le formulaire.
  • - method: permet d'indiquer si on récupère les données ou si on les envoie. Exemple: lorsque l'on s'inscrit sur un site, on envoie des données vers la base de données (BDD). Lorsque l'on se connecte, on récupère et on vérifie des données qui se trouvent déjà dans une BDD. Il existe 2 méthodes:
    • -> method="get": les informations sont passées et visible dans la barre d'adresse
    • -> method="post": envoie les données dans le corps de la requête sans passer par la barre d'adresse, c'est la méthode la plus utilisée et la plus sécurisée.

Présentation des différentes balises de formulaire comprise dans un <form action="#" method="post" enctype="multipart/form-data" autocomplete="off">

enctype="multipart/form-data" -> on l'utilise quand on a un champ d'envoi de fichier dans le formulaire

Lorsque la valeur de l'attribut autocomplete est défini sur "off" cela signifie que les navigateurs ne doivent pas suggérer ou compléter automatiquement les valeurs des champs de formulaire basées sur l'historique de saisie de l'utilisateur


label désigne l'étiquette du champs, elle permet de définir et d'informer sur l'info attendu


input: élément de saisie de formulaire: son apparence repose grandement sur la valeur de l'attribut type qui l'accompagne:

  • - input avec type text -> zone de texte monoligne

Informations personnelles
Nom et prénom <input type="text" id="nom" name="nom" placeholder="Votre nom de famille" maxlength="15" required>

name -> pour récupérer la valeur en PHP (pas d'espace, pas d'accent ni de caractères spéciaux)

placeholder -> garder la place = exemple de contenu valable, informer l'utilisateur sur ce qui est attendu.

required -> indique que le champs doit être obligatoirement renseigné. C'est un attribut particulier: il n'a pas de valeur


maxlength -> indique le nombre de caractère maximum

date de naissance <input type="date">

type="date" -> permet d'afficher un calendrier pour choisir une date

age <input type="number">

type="number" -> permet d'entre seulement un nombre

<input type="radio">

Lorsque l'on fait des boutons de type radio cela signifie qu'un seul peut être coché. Naturellement, celui qui a l'attribut checked sera coché d'office.





telephone <input type="tel" pattern="^[0-9]{10}$">

type="tel" -> permet d'afficher le clavier chiffré pour les smartphones

pattern est une expression rationnelle à laquelle doit correspondre le texte saiaie pour être valide

photo <input type="file">
type="file" -> permet l'envoi d'un fichier
residence <select name="pays" id="pays"> + <optgroup label=""> + <option value="">

la balise select vient toujours avec des options: elle permet de faire un menu déroulant, on la retrouve souvent pour l'année de naissance ou encore le département dans lequel on vit

Informations de connexion
mot de passe <input type="password">

type="password" -> permet de masquer le mot de passe

email <input type="email">

type="email" -> permet à l'utilisateur de saisir une adresse mail, la valeur va être automatiquement validée et vérifiée
